Summary

Background Using devices such as Jawbone Up, Nike FuelBand, and Fitbit it is now possible to collect a large amount of data about personal activity relatively inexpensively. These type of devices are part of the quantified self movement â€“ a group of enthusiasts who take measurements about themselves regularly to improve their health, to find patterns in their behavior, or because they are tech geeks. One thing that people regularly do is quantify how much of a particular activity they do, but they rarely quantify how well they do it. In this project, the goal will be to use data from accelerometers on the belt, forearm, arm, and dumbell of 6 participants. They were asked to perform barbell lifts correctly and incorrectly in 5 different ways. More information is available from the website here: http://groupware.les.inf.puc-rio.br/har (see the section on the Weight Lifting Exercise Dataset).

The Steps

Following the relative importance of steps for building a machine learning algorithm, i will start first by presenting the goal , then one of the most important steps wich is the data , then the features and finally the choice of the model

The question - Goal

The goal of the project is to build a prediction model to predict the outcome 'classe' variable indicating the manner in which these participants did their exercises . Any of the other variables can be used to predict with. Report should be created describing how the model was built , how cross validation was used, what the expected out of sample error is expected to be, and the reasons of the choices. The prediction model will be used to predict 20 different test cases.

The Model

Our data are used to predict five categories in classe outcome ( A, B, C, D, E) so we will need a supervised multi class classifier . If we assume that any small bad deviation of acceleration ( by belt, arm, forearm , dumbbell) in any direction (x,y,z) leads to a bad result then there should be for any of these movements ( accelerations) values in a decision tree that should fit the expected result. This is why a Random Forest should perform well if mtry is correctly fitted. Let us try with sifferent parameters number of folds : 5 repeats : 3 ntree : 100 trees 

control <- trainControl(method="repeatedcv", number=5, repeats=3)
metric <- "Accuracy"
modelRF <- train(classe~., data=training, method="rf", metric=metric,  trControl=control,ntree=100)
